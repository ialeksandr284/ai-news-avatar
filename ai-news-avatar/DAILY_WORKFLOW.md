# Daily Workflow

## Why This Exists
This file is the day-to-day runbook.
Use it when starting a fresh session so the workflow does not depend on remembering previous chats.

## Daily Run
1. Pull fresh stories from approved sources
2. Pick 1-3 candidates only
3. Rank candidates by:
   - clarity
   - novelty
   - practical impact
   - hook potential
4. Research the winner:
   - find the original source
   - verify the core claim
   - gather one supporting source if needed
5. Create a factual brief for the winner
6. Write one Russian short script
7. Create title and publish text
8. Run pre-publish review
9. Render the avatar video
10. Publish
11. Log the release

## Pre-Publish Review
Check:
- is the story real and sourced
- is the first sentence strong enough
- is the value clear immediately
- is the script too abstract
- is the title specific enough
- is the runtime likely under control
- does the render look acceptable

Reject if:
- the hook is weak
- the wording is generic
- the visual looks broken
- the story is too thin
- the story is not verified strongly enough

## Post-Publish Review
Review at:
- ~2 hours
- ~24 hours
- ~72 hours
- ~7 days

Collect:
- views
- engaged views
- average view duration
- average view percentage
- likes
- comments
- shares
- subscribers gained

## Interpretation Rules
- high views + low engagement = packaging worked, content depth weak
- low views + decent retention = story or title packaging weak
- high retention + low initial pickup = improve first line and title
- consistently weak performance on one topic type = reduce that topic cluster
- repeated wins on one angle = scale that angle carefully

## End of Day Update
After each publishing day:
1. Append releases to `logs/content_log.md`
2. Append insights to `logs/experiments_log.md`
3. If prompts changed, append to `logs/prompt_changes.md`
4. If channel strategy changed, update `SYSTEM.md`
